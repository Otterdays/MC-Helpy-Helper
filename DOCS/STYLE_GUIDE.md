# STYLE GUIDE

## Documentation Conventions

- Use top-down updates: newest notes appear first in active sections.
- Never delete prior notes; amend with dated notes when needed.
- Keep paths in forward-slash style in docs and code snippets.

## Trace Tag Format

- Java comments may use: `// [TRACE: <doc-file>.md]`
- Use trace comments sparingly and only where behavior is non-obvious.

## Comment Rules

- Prefer WHY comments over WHAT comments.
- Use prefixes consistently: `TODO:`, `FIXME:`, `NOTE:`.
